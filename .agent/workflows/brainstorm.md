# Brainstorm Workflow

## Purpose

Explore possibilities before committing to implementation. This workflow helps discover requirements, consider alternatives, and make informed decisions.

## Process

### Step 1: Understand the Goal

- What problem are we solving?
- Who benefits from this?
- What constraints exist?

### Step 2: Generate Options

- List at least 3 different approaches
- Include unconventional ideas
- Do not filter yet

### Step 3: Evaluate Trade-offs

For each option:
- Pros
- Cons
- Effort estimate
- Risk level

### Step 4: Narrow Down

- Eliminate clearly unsuitable options
- Identify top 2-3 candidates
- Note what information is missing

### Step 5: Recommend

- State preferred approach
- Explain reasoning
- Identify next steps

## Output Format

```markdown
## Goal
[Problem statement]

## Options Considered

### Option A: [Name]
- Description: ...
- Pros: ...
- Cons: ...
- Effort: Low/Medium/High

### Option B: [Name]
...

## Recommendation

[Preferred option] because [reasons].

## Next Steps

1. ...
2. ...

## Questions to Resolve

- ...
```

## When to Use

- Starting a new feature
- Facing a technical decision
- Unclear requirements
- Multiple valid approaches exist
